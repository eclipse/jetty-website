Example: Logging with Apache Log4j

This page describes how to configure Jetty for logging with Apache Log4j, which involves using Slf4j and the Slf4j binding layer for Log4j.

The Required JARs

You need to download three JARs, which you can do by clicking the following links:

slf4j-log4j12-1.6.6.jar

The Slf4j binding layer for Log4j.

slf4j-api-1.6.6.jar

The basic Slf4j API that Jetty's Slf4jLog implementation uses.

log4j-1.2.7.jar

Theversion of log4j that slf4j-log4j12-1.6.6 was written against.

Configuring Server Classpath

For the Server to use these new JARs you need to put them in place and tell Jetty to load them on each startup. Create a ${jetty.home}/lib/logging/ directory and place the three JAR files you downloaded there.

[jetty-distribution-9.1.5.v20140505]$ ls -l lib/logging/
total 520
-rw-rw-r-- 1 jetty jetty 489884 Mar 11 15:40 log4j-1.2.17.jar
-rw-rw-r-- 1 jetty jetty  26176 Mar 11 15:40 slf4j-api-1.6.6.jar
-rw-rw-r-- 1 jetty jetty   9711 Mar 11 15:40 slf4j-log4j12-1.6.6.jar

Next you edit the ${jetty.home}/start.ini to get Jetty to load these logging files into the Server classpath. Check for OPTIONS line with logging defined. If you can't find an OPTIONS entry with logging, add one at the end of the file.

[jetty-distribution-9.1.5.v20140505]$ grep logging start.ini 
# Adding lib/logging to server classpath
OPTIONS=logging

Now verify that the server classpath loads the three JARs.

[jetty-distribution-9.1.5.v20140505]$ java -jar start.jar --version | grep logging
Active Options: [Server, client, ext, jmx, jsp, logging, resources, websocket]
18:               1.2.17 | ${jetty.home}/lib/logging/log4j-1.2.17.jar
19:                1.6.6 | ${jetty.home}/lib/logging/slf4j-api-1.6.6.jar
20:                1.6.6 | ${jetty.home}/lib/logging/slf4j-log4j12-1.6.6.jar
